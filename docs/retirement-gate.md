# Retirement Gate

The retirement gate is the final decision point. The old Mac should remain available until the new Mac has passed validation and the user explicitly approves retirement.

## Do Not Retire Yet If

- Important folders are still `missing` or `blocked`.
- Cloud sync is still pending.
- External drives have not been checked.
- Critical apps cannot open.
- A password manager, email account, or browser profile is not accessible.
- Developer projects have not passed their agreed smoke tests.
- The user is unsure which Mac has the current copy of key work.

## Retirement Packet

Prepare a short packet with:

1. What is confirmed on the new Mac.
2. What is user-confirmed but not directly checked.
3. What is inferred.
4. What is missing.
5. What is blocked.
6. What can wait.
7. What the user is approving.

Use `templates/retirement-approval.md` for this packet.

## Approval Language

Use plain approval language. The user should know exactly what happens next and what risk remains.

Example:

```text
I approve retiring the old Mac after the checks listed as confirmed. I understand that items listed as missing or blocked have not been proven on the new Mac.
```

## After Approval

Only after approval should the user consider next steps such as signing out, erasing, trading in, donating, storing, or repurposing the old Mac. Follow Apple and service-provider instructions for account removal and device erase.

Next step: complete `templates/retirement-approval.md` and keep the old Mac unchanged until the user approves the packet.
