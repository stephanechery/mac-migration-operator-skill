# Launch Notes

These notes turn Mac Migration Operator Skill from an open repo into a small paid support offer. They are written for a careful first launch, not a high-volume sales push.

## Current Status

- GitHub repository is public.
- SkillsLLM submission is under review.
- Capafy was reported as added by the maintainer, pending public indexing.
- ClaudeSkills.info is blocked by a generic submission error after working Google sign-in and failed GitHub URL plus ZIP attempts.
- skills.sh can install the Skill, but search indexing has not shown it yet.

## Launch Goal

Get the first 3 to 5 real users to try the free Skill or book a paid review. Use those sessions to improve the workflow, collect plain-language feedback, and decide whether to keep selling a guided service.

## Best First Offer

Start with:

```text
Guided Migration Validation, $249
```

Why this tier first:

- It has a clear buyer pain after Migration Assistant finishes.
- It avoids taking full responsibility for the live transfer.
- It creates useful evidence: validation checklist, gap list, and retirement decision.
- It can be delivered remotely with low setup.

Offer promise:

```text
I will help you validate whether your new Mac is ready to become your main machine and what still needs attention before you retire the old one.
```

## Launch Sequence

1. Publish the GitHub repo and free Skill. Already done.
2. Submit to Skill directories and marketplaces. SkillsLLM submitted, Capafy pending, ClaudeSkills blocked.
3. Add a simple offer page once price, booking, and payment method are approved.
4. Post the launch on LinkedIn and X.
5. Offer 3 early-review slots at an introductory price or with extra async support.
6. Capture questions, blockers, and repeated buyer concerns.
7. Update the Skill and the offer page with what users actually need.

## Proof to Collect

For each early user, record:

- Migration type.
- What they were most worried about.
- What the Skill helped them catch.
- Time saved or risk reduced.
- Whether they would pay for guided support.
- Exact wording they use to describe the value.

Do not record private file names, account names, medical, legal, financial, HR, or personal data in public notes.

## Approval Needed Before Revenue

Bring the maintainer in before:

- Publishing paid prices.
- Opening paid booking slots.
- Connecting a payment provider.
- Adding payout or tax details.
- Promising response times.
- Sending direct sales messages from personal or business accounts.
