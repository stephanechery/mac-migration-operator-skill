# Cloud Drives

Cloud drives can make migration easier, but they can also hide missing data behind sync delays or online-only placeholders.

## Identify Cloud Services

Record every cloud drive the user depends on:

- iCloud Drive
- Google Drive
- OneDrive
- Dropbox
- Box
- Any work or school sync tool

For each service, record the account label, sync status, and top folders that matter.

## Before Transfer

1. On the source Mac, confirm the cloud app is signed in.
2. Check whether important folders are fully downloaded or online-only.
3. Wait for pending uploads to finish before freezing the source Mac.
4. Record the source state in the proof ledger.

## On the New Mac

1. Sign in to each cloud service.
2. Confirm the right account is active.
3. Confirm the expected top folders appear.
4. Open a few important files.
5. Check whether files are downloaded or still online-only.

## Placeholder Files

An online-only file may appear in Finder but not be fully downloaded. Do not mark it `confirmed` until it opens successfully or the cloud app confirms it is available offline.

## Conflict Files

If the cloud service creates conflict copies, do not delete either copy during validation. Record both paths, compare modified dates, and ask the user which copy is trusted.

## Slow Sync

If sync is slow, mark the item `blocked` with the date and time. Check again after the cloud app says sync is complete.

Next step: choose one cloud drive and confirm the three most important folders open on the new Mac.
