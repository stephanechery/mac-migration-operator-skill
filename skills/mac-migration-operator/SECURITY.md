# Security

This project is designed for careful migration work. Treat every Mac migration as high-stakes.

Do not submit issues, logs, screenshots, pull requests, or support notes that include passwords, tokens, private keys, API keys, full identity documents, client records, medical records, tax records, private financial records, or other sensitive personal data.

Helper scripts must be read-only unless a future change clearly marks a script as write-capable and requires explicit user approval before it runs.

Do not print or store secrets as evidence. It is acceptable to record that access appears active, that a command reports a signed-in state, or that an app opened after sign-in. It is not acceptable to record secret values.

Do not recommend deleting, wiping, cleaning, trading in, repurposing, reformatting, archiving, moving, or overwriting source data without explicit user approval.

Report security concerns privately through GitHub Security Advisories when the repository is published.

Next step: if you find a security concern, prepare a private report with the affected file, the risk, and a safe reproduction that does not include secret values.
